## 2. システムアーキテクチャ

### 2.1 全体構成

本システムは **クライアント・サーバー型** のWebアプリケーションである。

```
┌─────────────────────────────────────────┐
│          クライアント（ブラウザ）            │
│  ┌──────────────────────────────────┐   │
│  │  HTML / CSS / JavaScript (SPA風)  │   │
│  └──────────────────────────────────┘   │
└────────────────┬────────────────────────┘
                 │ HTTP REST API (port 3000)
┌────────────────▼────────────────────────┐
│          サーバー（Node.js）              │
│  ┌──────────────────────────────────┐   │
│  │  Express.js（Webサーバー＋API）   │   │
│  └──────────────┬───────────────────┘   │
│  ┌──────────────▼───────────────────┐   │
│  │  In-Memoryデータストア（配列）     │   │
│  └──────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

## 4. 機能分割

| 機能ID | 機能名 | 担当レイヤー |
|--------|--------|------------|
| FR-01 | Todo追加 | フロント（入力UI） + サーバー（POST /api/todos） |

## 5. データフロー

### 5.1 Todo追加フロー

```
ユーザー入力
    → [フロント] バリデーション（空文字チェック）
    → [フロント] POST /api/todos { text }
    → [サーバー] バリデーション
    → [サーバー] todos配列に追加 { id, text, completed: false }
    → [サーバー] 201 Created + 追加データ返却
    → [フロント] GET /api/todos で再取得 → DOM再描画
```

### 5.2 完了切替フロー

```
チェックボックス操作
    → [フロント] PATCH /api/todos/:id
    → [サーバー] 該当Todoの completed を反転
    → [サーバー] 200 OK + 更新データ返却
    → [フロント] GET /api/todos で再取得 → DOM再描画
```

### 5.3 削除フロー

```
削除ボタン押下
    → [フロント] DELETE /api/todos/:id
    → [サーバー] 該当Todoを配列から除去
    → [サーバー] 204 No Content
    → [フロント] GET /api/todos で再取得 → DOM再描画
```

## 6. APIエンドポイント一覧

| メソッド | パス | 概要 |
|---------|------|------|
| GET | /api/todos | Todo全件取得 |
| POST | /api/todos | Todo新規追加 |
| PATCH | /api/todos/:id | 完了状態切替 |
| DELETE | /api/todos/:id | Todo削除 |

---

## 7. データモデル

### Todoオブジェクト

| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | number | 自動採番の一意識別子（1始まりの整数） |
| text | string | タスク名（空文字・空白不可） |
| completed | boolean | 完了フラグ（初期値: false） |

---

## 8. エラー処理方針

| エラー種別 | HTTPステータス | 対応 |
|----------|--------------|------|
| 空テキスト登録 | 400 Bad Request | エラーメッセージ返却 |
| 存在しないIDへの操作 | 404 Not Found | エラーメッセージ返却 |
| サーバー内部エラー | 500 Internal Server Error | エラーメッセージ返却 |
