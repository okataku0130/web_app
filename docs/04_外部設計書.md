## 2. 画面設計

### 2.1 画面一覧

| 画面ID | 画面名 | URL | 概要 |
|--------|--------|-----|------|
| SCR-01 | Todoリスト画面 | http://localhost:3000/ | アプリの唯一の画面。全機能を提供。 |

### 2.2 画面レイアウト（SCR-01）

```
┌────────────────────────────────────┐
│           Todoリスト                │  ← タイトル（h1）
├──────────────────────────┬─────────┤
│  テキスト入力欄             │ 追加ボタン │  ← 入力行
├────────────────────────────────────┤
│  N件中 M件完了                       │  ← 進捗統計（Todoが1件以上の場合）
├────────────────────────────────────┤
│ ☐  タスク名1                    [✕] │
│ ☑  タスク名2（打消し線）          [✕] │  ← Todoリスト
│ ☐  タスク名3                    [✕] │
│  ...                               │
└────────────────────────────────────┘
※ Todo 0件時: 「タスクがありません」を表示
```

### 2.3 UIコンポーネント仕様

| コンポーネント | 種類 | 動作 |
|-------------|------|------|
| タイトル | h1テキスト | 「Todoリスト」固定表示 |
| テキスト入力欄 | `<input type="text">` | プレースホルダー「新しいタスクを入力...」。フォーカス時ボーダー色変化 |
| 追加ボタン | `<button>` | クリックでTodo追加。ホバーで背景色変化 |
| 進捗統計 | テキスト | 「N件中 M件完了」。0件時は非表示 |
| チェックボックス | `<input type="checkbox">` | クリックで完了/未完了を切替 |
| タスク名 | `<span>` | 未完了: 通常表示、完了: 打ち消し線＋グレー |
| 削除ボタン | `<button>` | 「✕」ボタン。クリックでTodo削除 |
| 空表示 | テキスト | 「タスクがありません」Todo0件時のみ表示 |

### 2.4 カラーパレット

## 3. 操作仕様

### 3.1 Todo追加

| 操作 | 結果 |
|------|------|
| テキスト入力 → 「追加」ボタンクリック | Todoを追加。入力欄をクリア |
| テキスト入力 → Enterキー押下 | 同上 |
| 空欄のまま追加操作 | 何も起きない（追加されない） |

### 3.2 完了/未完了切替

| 操作 | 結果 |
|------|------|
| 未完了タスクのチェックボックスをクリック | 完了状態へ変更（打ち消し線表示） |
| 完了タスクのチェックボックスをクリック | 未完了状態へ変更（通常表示） |

### 3.3 Todo削除

| 操作 | 結果 |
|------|------|
| 「✕」ボタンをクリック | 対象Todoを即時削除 |

## 4. API外部仕様

### 4.1 GET /api/todos

**概要:** 登録済みTodo全件を取得する

**リクエスト:**
```
GET /api/todos HTTP/1.1
Host: localhost:3000
```

**レスポンス（成功 200 OK）:**
```json
[
  { "id": 1, "text": "買い物をする", "completed": false },
  { "id": 2, "text": "メールを返信する", "completed": true }
]
```

**レスポンス（0件 200 OK）:**
```json
[]
```

---

### 4.2 POST /api/todos

**概要:** 新しいTodoを追加する

**リクエスト:**
```
POST /api/todos HTTP/1.1
Host: localhost:3000
Content-Type: application/json

{ "text": "新しいタスク" }
```

**レスポンス（成功 201 Created）:**
```json
{ "id": 3, "text": "新しいタスク", "completed": false }
```

**レスポンス（バリデーションエラー 400 Bad Request）:**
```json
{ "error": "テキストを入力してください" }
```

---

### 4.3 PATCH /api/todos/:id

**概要:** 指定IDのTodoの完了状態を反転する

**リクエスト:**
```
PATCH /api/todos/1 HTTP/1.1
Host: localhost:3000
```

**レスポンス（成功 200 OK）:**
```json
{ "id": 1, "text": "買い物をする", "completed": true }
```

**レスポンス（未存在 404 Not Found）:**
```json
{ "error": "見つかりません" }
```

---

### 4.4 DELETE /api/todos/:id

**概要:** 指定IDのTodoを削除する

**リクエスト:**
```
DELETE /api/todos/1 HTTP/1.1
Host: localhost:3000
```

**レスポンス（成功 204 No Content）:**
```
（ボディなし）
```

**レスポンス（未存在 404 Not Found）:**
```json
{ "error": "見つかりません" }
```

## 5. 入出力項目定義

### 5.1 入力項目

| 項目名 | 入力欄 | データ型 | 制約 | 説明 |
|--------|--------|---------|------|------|
| タスク名 | テキスト入力欄 | string | 必須・空文字不可・前後空白はトリム | 登録するTodoのタスク名 |

### 5.2 出力項目（Todoリスト）

| 項目名 | 表示場所 | データ型 | 説明 |
|--------|---------|---------|------|
| id | 非表示（内部利用） | number | Todo一意識別子 |
| text | タスク名表示欄 | string | タスク名 |
| completed | チェックボックス・スタイル | boolean | 完了状態 |
| 総件数 | 進捗統計 | number | 登録中のTodo総数 |
| 完了件数 | 進捗統計 | number | 完了済みTodo数 |
