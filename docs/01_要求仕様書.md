## 1. 目的

本文書は、Todoリスト作成アプリに対するステークホルダーの要求を整理し、開発チームへの伝達を目的として作成する。

---

## 2. プロジェクト概要

### 2.1 背景・課題

日常業務や個人の作業管理において、タスクの把握・優先順位付けが属人的になりがちである。シンプルかつ直感的に操作できるタスク管理ツールを提供することで、作業効率の向上を図る。

### 2.2 目標

- ブラウザから即座にアクセスできるWebベースのTodoリストを提供する
- 特別なインストール不要で利用できる
- タスクの追加・完了・削除を最小限の操作で実現する

---

## 3. ステークホルダー

| 区分 | 役割 |
|------|------|
| エンドユーザー | Webブラウザからアプリを利用する一般ユーザー |
| 開発者 | アプリの設計・実装・保守を行う担当者 |
| 管理者 | サーバーの運用・管理を行う担当者 |

---

## 4. ユーザー要求

### 4.1 機能要求

| No. | 要求内容 |
|-----|---------|
| UR-01 | タスクを入力してリストに追加したい |
| UR-02 | タスクが完了したらチェックして完了状態にしたい |
| UR-03 | 不要なタスクを削除したい |
| UR-04 | 現在のタスク一覧を一目で確認したい |
| UR-05 | 完了済みタスク数と総タスク数を把握したい |
| UR-06 | キーボードだけで操作を完結させたい（Enterキーで追加） |

### 4.2 非機能要求

| No. | 要求内容 |
|-----|---------|
| UR-07 | PCのブラウザから利用できること |
| UR-08 | 操作に対してすぐに画面が反応すること |
| UR-09 | シンプルで見やすいUIであること |
| UR-10 | サーバーが動いている間はデータが保持されること |

---

## 5. 制約条件

| No. | 制約内容 |
|-----|---------|
| CO-01 | 実装言語はNode.js（JavaScript）とする |
| CO-02 | データベースは使用せず、サーバーメモリ上でデータを管理する（初期版） |
| CO-03 | 対応ブラウザはモダンブラウザ（Chrome / Edge 最新版）とする |
| CO-04 | ポート番号は3000番を使用する |

---

## 6. 前提条件

- Node.js（v18以上）がサーバーにインストールされていること
- npmが使用可能であること
- ポート3000番が開放されていること

---

## 7. 用語定義

| 用語 | 定義 |
|------|------|
| Todo | ユーザーが登録するタスク（作業項目） |
| 完了状態 | チェックボックスにチェックが入り、打ち消し線が表示された状態 |
| 未完了状態 | チェックボックスにチェックが入っていない初期状態 |
