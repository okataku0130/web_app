## 3. サーバー側詳細設計（server.js）

### 3.1 モジュール・変数

| 名前 | 種別 | 型 | 初期値 | 説明 |
|------|------|-----|--------|------|
| `express` | 定数 | module | require('express') | Expressフレームワーク |
| `path` | 定数 | module | require('path') | パス解決ユーティリティ |
| `app` | 定数 | Express app | express() | Expressアプリケーション |
| `PORT` | 定数 | number | 3000 | 待受ポート番号 |
| `todos` | 変数 | Array\<Todo\> | [] | Todoデータストア（In-Memory） |
| `nextId` | 変数 | number | 1 | 次のTodoに割り当てるID |

### 3.2 Todoオブジェクト型定義

```
Todo {
  id:        number   // 1から始まる自動採番の整数
  text:      string   // タスク名（空白トリム済み）
  completed: boolean  // 完了フラグ（初期値: false）
}
```

### 3.3 ミドルウェア設定

| ミドルウェア | 設定 | 目的 |
|------------|------|------|
| `express.json()` | グローバル | リクエストボディのJSONパース |
| `express.static()` | `public/` ディレクトリ | 静的ファイル配信 |

### 3.4 APIルート詳細

#### GET /api/todos

```
処理フロー:
  1. todos配列をそのままJSON形式でレスポンス
  2. ステータスコード: 200 OK

疑似コード:
  app.get('/api/todos', (req, res) => {
    res.json(todos)
  })
```

#### POST /api/todos

```
処理フロー:
  1. req.body.text を取得
  2. text が存在しない or trim()後が空文字の場合
     → 400 Bad Request + { error: 'テキストを入力してください' }
  3. Todoオブジェクト生成: { id: nextId++, text: text.trim(), completed: false }
  4. todos配列末尾に追加
  5. 201 Created + 生成したTodoオブジェクトをJSON返却

バリデーション:
  - text: 必須。undefined / null / 空文字 / 空白のみ → エラー
  - text.trim(): 前後空白を除去して保存
```

#### PATCH /api/todos/:id

```
処理フロー:
  1. req.params.id を parseInt() で数値変換
  2. todos.find() で該当Todoを検索
  3. 見つからない場合 → 404 Not Found + { error: '見つかりません' }
  4. todo.completed を反転（true↔false）
  5. 200 OK + 更新後のTodoオブジェクトをJSON返却
```

#### DELETE /api/todos/:id

```
処理フロー:
  1. req.params.id を parseInt() で数値変換
  2. todos.findIndex() で該当Todoのインデックスを検索
  3. インデックスが -1 の場合 → 404 Not Found + { error: '見つかりません' }
  4. todos.splice(index, 1) で配列から削除
  5. 204 No Content を返却（ボディなし）
```

## 4. クライアント側詳細設計（public/index.html）

### 4.1 HTML構造

```
body
└── div.container
    ├── h1                     ← タイトル
    ├── div.input-row
    │   ├── input#todoInput    ← テキスト入力欄
    │   └── button.add-btn     ← 追加ボタン (onclick="addTodo()")
    ├── div#stats              ← 進捗統計テキスト
    ├── ul#todoList            ← Todoリスト（動的生成）
    └── p#empty                ← 0件時メッセージ
```

### 4.2 JavaScript関数詳細

#### fetchTodos()

```
概要: サーバーからTodo一覧を取得しDOMを更新する
引数: なし
戻り値: なし（async）

処理:
  1. GET /api/todos をfetch()で呼び出す
  2. レスポンスをJSON配列としてパース
  3. render(todos) を呼び出す
```

#### addTodo()

```
概要: 入力欄のテキストをサーバーへ送信し、Todoを追加する
引数: なし
戻り値: なし（async）

処理:
  1. input#todoInput の value を取得し trim()
  2. 空文字の場合は早期リターン（処理中断）
  3. POST /api/todos を fetch() で呼び出す
     - Content-Type: application/json
     - body: JSON.stringify({ text })
  4. 入力欄の value を空文字にリセット
  5. fetchTodos() を呼び出す
```

#### toggleTodo(id)

```
概要: 指定IDのTodoの完了状態を切り替える
引数: id (number) - 対象TodoのID
戻り値: なし（async）

処理:
  1. PATCH /api/todos/${id} を fetch() で呼び出す
  2. fetchTodos() を呼び出す
```

#### deleteTodo(id)

```
概要: 指定IDのTodoを削除する
引数: id (number) - 対象TodoのID
戻り値: なし（async）

処理:
  1. DELETE /api/todos/${id} を fetch() で呼び出す
  2. fetchTodos() を呼び出す
```

#### render(todos)

```
概要: Todo配列からDOMを生成し画面を更新する
引数: todos (Array<Todo>) - 表示するTodo一覧
戻り値: なし

処理:
  1. ul#todoList の innerHTML を空にする
  2. todos.length === 0 の場合:
     - p#empty を表示 (display: 'block')
     - div#stats を空文字に
     - 早期リターン
  3. todos.length > 0 の場合:
     - p#empty を非表示 (display: 'none')
     - completed数をfilter().lengthで算出
     - div#stats に「N件中 M件完了」を設定
     - todos.forEach() で各Todoの <li> を生成:
         <li>
           <input type="checkbox" [checked] onchange="toggleTodo(id)">
           <span class="[done]">escapeHtml(text)</span>
           <button class="del-btn" onclick="deleteTodo(id)">✕</button>
         </li>
     - 生成した <li> を ul#todoList に appendChild
```

#### escapeHtml(str)

```
概要: HTMLタグインジェクションを防ぐため特殊文字をエスケープする
引数: str (string) - エスケープ対象文字列
戻り値: string - エスケープ済み文字列

変換ルール:
  & → &amp;
  < → &lt;
  > → &gt;
```

### 4.3 イベントリスナー

| イベント | 対象要素 | 処理 |
|---------|---------|------|
| keydown（key === 'Enter'） | input#todoInput | addTodo() を呼び出す |
| DOMContentLoaded（初期化） | - | fetchTodos() を呼び出す |

### 4.4 CSSクラス定義

| クラス名 | 適用要素 | 効果 |
|---------|---------|------|
| `.container` | div | 白背景・角丸・シャドウ・最大幅500px |
| `.input-row` | div | flexレイアウト（入力欄+ボタンを横並び） |
| `.add-btn` | button | 紫系グラデーション背景・白文字 |
| `.del-btn` | button | 背景なし・赤文字（#fc8181） |
| `.done` | span | 打ち消し線・グレー文字（#a0aec0） |
| `.empty` | p | 中央揃え・グレー文字 |

## 5. セキュリティ考慮事項

| 項目 | 対策 |
|------|------|
| XSS対策 | DOM生成時に escapeHtml() でタスク名をエスケープ |
| 入力バリデーション | フロント・サーバー両側で空文字チェックを実施 |
| プロトタイプ汚染 | ユーザー入力をオブジェクトのキーとして使用しない |
